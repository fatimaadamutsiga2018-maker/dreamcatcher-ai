## 1. 模块概述

事件解读模块是 ClarityPath 的核心功能之一，允许用户输入任意具体事件，AI 结合当前时机（能量等级 + 时辰面板）给出针对性的解读建议。

与"今日能量"不同，事件解读聚焦于**特定决策的适配度分析**，帮助用户判断"现在做这件事"的时机是否合适。

---

## 2. 核心设计原则

### 2.1 三层架构

```
逻辑层 (Logic)     → 意图识别、时机评估、匹配解读角度
模板层 (Template)  → 五段式输出结构（快照→洞察→指导→行动→钩子）
护栏层 (Guardrails) → 语言约束、深度限制、安全边界
```

### 2.2 五不原则

1. **不预测结果** — 不说"你会成功/失败"
2. **不贴身份标签** — 能量状态是暂时的，不是"你是XX型人"
3. **不用算命术语** — 不用"冲煞""犯太岁"等词
4. **不给绝对指令** — 不说"你必须/千万不要"
5. **不制造焦虑** — 不用"错过就完了"等紧迫话术

## 3. 事件分类体系

| 类型 | 关键词触发 | 解读侧重点 | 推荐语气 |
|------|-----------|-----------|---------|
| **职业** | 工作、辞职、面试、升职、转行、创业 | 时机成熟度、风险缓冲、长期适配 | Striver（减压务实） |
| **关系** | 表白、分手、复合、结婚、冲突、合作 | 沟通窗口、情绪状态、双方节奏 | Reset Seeker（温柔恢复） |
| **财务** | 投资、买房、大额消费、债务、谈薪 | 风险承受、决策紧迫度、备选方案 | Explorer（理性探索） |
| **健康** | 手术、健身、作息调整、体检 | 身体节律、恢复周期、渐进策略 | Reset Seeker（温和渐进） |
| **创作** | 启动项目、发布作品、签约、投稿 | 灵感状态、完成度、反馈时机 | Explorer（好奇开放） |
| **学习** | 考证、转专业、留学、培训 | 专注度、投入产出、时间窗口 | Striver（目标导向） |
| **迁徙** | 搬家、移民、换城市、旅行 | 环境适应、准备度、告别-启程节奏 | Explorer（开放适应） |

## 4. 意图识别层

### 4.1 输入解析

```
用户输入: "{user_input}"

步骤1: 提取事件核心
- 用10个字以内概括这件事
- 判断事件类型（从分类表中选最接近的）

步骤2: 识别用户状态
- 用户是【已决定】还是【犹豫中】？
- 用户最担心的点是什么？（失败/时机/他人反应/准备不足/其他）

步骤3: 匹配解读深度
- 简单事件 → 标准五段式输出
- 复杂决策 → 增加"常见误区"和"备选路径"

输出格式（JSON）:
{
  "event_summary": "事件简述",
  "event_type": "类型",
  "user_stance": "decided/hesitant",
  "core_concern": "主要顾虑",
  "depth_level": "standard/deep"
}
```

### 4.2 复杂决策判断标准

以下情况触发深度模式：
- 涉及多个利益相关方（家庭、团队）
- 有明确的时间压力（deadline 在1周内）
- 财务影响较大（超过月收入3倍）
- 不可逆或难以撤销（辞职、分手、大额投资）

## 5. 时机评估逻辑

### 5.1 时辰面板 → NOW/WAIT/ADJUST 映射

| 时辰面板 | 时机判断 | 核心含义 | 行动建议 |
|---------|---------|---------|---------|
| 建 | NOW | 开始、建立 | 适合启动新事物 |
| 开 | NOW | 开放、开启 | 适合公开、沟通、签约 |
| 成 | NOW | 成就、完成 | 适合收尾、交付、庆祝 |
| 满 | NOW | 圆满、充盈 | 适合收获、享受成果 |
| 除 | ADJUST | 清除、整理 | 适合清理、结束旧事务 |
| 平 | ADJUST | 平稳、平衡 | 适合维护、保持现状 |
| 定 | ADJUST | 稳定、确定 | 适合规划、准备、等待 |
| 执 | ADJUST | 执行、坚持 | 适合持续推进已有计划 |
| 收 | ADJUST | 收敛、收获 | 适合盘点、总结、储蓄 |
| 闭 | WAIT | 关闭、休息 | 建议暂停、内省、恢复 |
| 危 | WAIT | 危机、谨慎 | 建议观望、避免冒险 |
| 破 | WAIT | 破坏、变革 | 建议等待、不要强行推进 |

### 5.2 能量等级 × 时机判断矩阵

| 能量等级 | NOW | ADJUST | WAIT |
|---------|-----|--------|------|
| **Tier 5** (Highly Favorable) | 🟢 绿灯，全力推进 | 🟡 黄灯，边做边优化 | 🟡 黄灯，可行动但需更多准备 |
| **Tier 4** (Favorable) | 🟢 绿灯，积极行动 | 🟡 黄灯，注意细节 | 🟡 黄灯，小步试探 |
| **Tier 3** (Moderate) | 🟡 黄灯，需要额外努力 | 🟡 黄灯，稳扎稳打 | 🟠 橙灯，建议暂缓 |
| **Tier 2** (Challenging) | 🟠 橙灯，风险较高 | 🟠 橙灯，先处理障碍 | 🔴 红灯，强烈建议等待 |
| **Tier 1** (Unfavorable) | 🔴 红灯，不建议 | 🔴 红灯，先恢复能量 | 🔴 红灯，完全等待 |

## 6. 输出模板

### 6.1 标准五段式

```markdown
## 事件解读 — {event_summary}

**当前时机**: {date} · {tier描述} · {almanac_panel} ({NOW/WAIT/ADJUST})

**事件快照**:
{1句话点出这件事的本质，让用户感到被理解}

**能量洞察**:
{2-3句话，结合tier和时机，分析当前做这件事的"天时"状态}
{如果用户是hesitant，先回应顾虑；如果是decided，给执行建议}

**时机指导**:
- ✅ 有利因素: {1-2点}
- ⚠️ 需要注意: {1-2点}
- ⏸️ 如果犹豫: {1个自我检视问题}

**你的微行动**:
{一个今天就能完成的、降低压力的小步骤}

**继续探索**:
{根据事件类型推荐下一步: 决策六爻 / 愿景板 / 能量记录 / 日记}
```

### 6.2 深度模式扩展

在标准模板基础上，在"时机指导"后增加：

```markdown
**常见误区**:
- {该情境下人们常犯的错误1}
- {该情境下人们常犯的错误2}

**备选路径**:
如果主要计划受阻，考虑：
- {替代方案1}
- {替代方案2}
```

## 7. 语气与 Archetype 映射

### 7.1 Explorer（探索者）

**适用场景**: 创作、财务、迁徙、新尝试

**语言特征**:
- 用"试试看""探索""发现"等词
- 强调可能性和学习
- 降低对"正确结果"的执念

**示例**:
> "这是一个值得探索的时机，无论结果如何，你都会对自己有更多了解。"

### 7.2 Striver（奋斗者）

**适用场景**: 职业、学习、目标导向的任务

**语言特征**:
- 承认努力，减轻压力
- 强调过程价值，不只关注结果
- 提供具体、可操作的步骤

**示例**:
> "你已经为此付出了很多努力。现在的关键是，不要让完美主义成为行动的阻碍。"

### 7.3 Reset Seeker（重启者）

**适用场景**: 关系、健康、犹豫期、低谷期

**语言特征**:
- 温柔、缓慢、不催促
- 强调自我照顾和恢复
- 给足时间和空间

**示例**:
> "在做出任何决定之前，先问问自己：我现在的状态，是否足以支撑这个决定？"

## 8. 跟进对话护栏

### 8.1 问题类型应对表

| 用户问法 | AI回应策略 | 禁止 |
|---------|-----------|------|
| "再详细说说" | 深化隐喻，举1个具体例子，解释tier含义 | 编造新数据 |
| "我应该做吗？" | 重申时机评估，说"这倾向于..."，让用户自己决定 | 给绝对答案 |
| "如果我不听呢？" | 提醒这是参考，不是规则，尊重用户选择 | 吓唬用户 |
| "帮我算算成功率" | 解释我们不预测结果，只提供时机参考 | 给百分比 |
| "哪天最好？" | 解释单日能量是参考，重要决策看整体趋势 | 翻黄历式回答 |
| "为什么是这个结果？" | 解释能量等级和时辰面板的映射逻辑 | 用玄学术语解释 |
| 连续追问 >2轮 | 建议记录到愿景板或日记，或转其他模块 | 无限纠缠 |
| 涉及危机/自伤 | 立即建议寻求专业帮助，提供资源链接 | 继续"解读" |

### 8.2 对话深度限制

- **第1轮**: 完整五段式输出
- **第2轮**: 针对具体问题的深化回应
- **第3轮起**: 必须引导至记录/其他模块，或总结结束

**第三轮标准话术**:
> "我们已经聊了不少。建议你把这个思考记录到【愿景板】，或者稍后再来看看是否有新的感受。需要我帮你打开记录界面吗？"

## 10. 系统 Prompt（完整版）

意图识别 + 主 prompt + 输出 prompt 三步：  
1. 列出 `event_summary/event_type/user_stance/core_concern/depth_level`（JSON）。  
2. 主 prompt 结合角色/原则/时机/事件生成 Snapshot→Avoid。  
3. 语言盾：用户用中文→中文回应，默认英语。

### 10.1 测试 prompt 参考
- 基础版（辞职）— Striver 语气、Tier4+NOW、五段式输出。  
- 完整版（分手）— 恢复者语气，带意图识别、常见误区、备选路径。  
- 多场景测试— Tier+NOW 下的创作/健身/财务/迁徙等短输出。  
- 护栏测试— 问“哪只股票”“成功率”“具体吉日”，AI 必须拒绝具体预测。

### 10.2 预期检查清单
- 无“你会成功/失败”、无身份标签、无算命术语、无绝对指令。  
- 语气贴合 Archetype，Micro Action 可今日完成。  
- 追问超2轮后，引导记录或跳转至其他模块。  
- 敏感问法一律用“时机参考”拒绝预测。

## 12. 与 `AI-Interpretation-Standard-V1.md` 的协同

`AI-Interpretation-Standard-V1.md` 同步提供了更简洁的信念/能量等级/时机/输出结构标准，适合做快速校验或复用在其他模块。当前文档和那份标准可以互为补充：

- 本文档聚焦事件解读一线流程、分类、意图、Archetype 语气和 UI 映射。
- 标准文档则提供统一的五级能量表、三类时机、意图轮廓、五段式结构说明以及示例+附录，适合各模块共用。

建议：在后续迭代中把 `AI-Interpretation-Standard-V1.md` 作为“通用规则集”，本页 `Event Interpretation` 文档引用其中的能源等级/ timing cue/ answer structure 说明，并保持双向更新。当 prompt 或 guardrail 变更时，先在标准文档里修正，再同步到事件模块，保证一致性。

## 附录

### A. 快速参考卡

```
【事件解读流程】
1. 接收用户输入 → 意图识别 → 输出JSON
2. 获取当前时机 → 能量等级 + 时辰面板
3. 评估适配度 → 查矩阵表 → 确定灯色
4. 选择语气 → 根据类型和状态选Archetype
5. 生成输出 → 按五段式填充
6. 等待跟进 → 查护栏表 → 限制深度
```

### B. 与现有模块的关系

```
ClarityPath 功能矩阵
├── 今日能量 (Daily Energy) ──→ 通用状态检查
├── 事件解读 (Event Interpretation) ──→ 特定决策分析 ★本模块
├── 决策六爻 (Decision Hexagram) ──→ 深度决策辅助
├── 愿景板 (Vision Board) ──→ 目标可视化与追踪
└── 能量记录 (Energy Journal) ──→ 长期模式观察
```

事件解读是承上启下的核心模块：
- 从"今日能量"获取基础时机数据
- 可引导至"决策六爻"做更深入分析
- 鼓励用户记录到"愿景板"或"能量记录"
